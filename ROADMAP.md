# Roadmap

## Done

- **BPM ingestion** — `bpm_source` column on `tracks`, Beatport (strict `__NEXT_DATA__` parse + exact artist+title match, half-tempo doubling for DnB) + GetSongBPM lookup, manual-tap residual; aubio dropped from the ingestion path (unreliable on breakbeat). ~892 tracks tagged in DB and written back to file metadata via mutagen (TBPM / tmpo / BPM).
- **Playlist generator** — `scripts/generate_run_playlists.py`: DB selection or `--from-tsv` re-render, artist-varied picking with duration tolerance, parallel retempo, crossfade mix, mp4 wrap with cover image, `<bpm>bpm/<mins>mins/` subfolder output, `--start-n` for numbering extensions without colliding with existing outputs.
- **Playlist extension** — `scripts/extend_playlist_tsvs.py`: tops up a short ramp tsv to a target duration by picking fresh tracks from the same BPM pool, respecting artist variety within the extended list and a global exclusion list of tracks already used in other playlists at that BPM.
- **Tracklist sidecars** — `generate_run_playlists.py` emits `{slug}_tracklist.txt` next to each mp4 with timestamps adjusted for crossfade overlap, ready to paste into YouTube descriptions as chapter markers. `scripts/write_tracklists.py` backfills tracklists for existing mixes by probing retempoed `_work` mp3s instead of re-rendering.
- **Cover overlay** — `tovideo.sh` accepts `-b BPM` / `-d MIN` and burns BPM + runtime badges into the bottom-right of the cover via ffmpeg `drawtext` (font overridable via `TOVIDEO_FONT`). `generate_run_playlists.py` passes target BPM and duration through automatically; `build-run-playlist.sh` exposes `-b` / `-m` pass-through flags. Makes per-mix YouTube thumbnails distinguishable at a glance.
- **Beat-clarity filter** — `scripts/analyse_beat_clarity.py` populates `music.db` with `groove_full/half/delta`, `grid_tightness`, `kick_prominence` and a `beat_review_flag` for each DnB-range track using librosa. Half-step tracks (`groove_delta < 0.05`) auto-exclude with `run_exclude_reason='half-step'`; off-grid/less-prominent tracks get surfaced via the review flag for manual triage. `scripts/probe_beat_clarity.py` is the diagnostic spike; `scripts/reject_track.py` is the manual exclude CLI (`<id> "reason"`, `--unexclude`, `--list`, `--review`, `--m3u` export). `scripts/audit_shipped_playlists.py` cross-checks shipped `*_tracklist.txt` against current exclusions. Initial run on 890 DnB tracks (97s, 4 workers) flagged 57 half-step + 184 scatty (all binned).
- **Traktor BPM import (v1)** — `scripts/import_traktor.py` parses `collection.nml` (Traktor Pro 3 export) and populates `tracks.traktor_bpm`, `traktor_key`, `traktor_beatgrid_ms`, `traktor_imported_at`. Additive columns, no overwrite of existing `bpm`. Path reconstruction handles Traktor's `/:` DIR separator. First run: 1917/1953 matched (98.2%), 1043 pure gains (no prior BPM), 690 exact agreement, 184 disagreements — traced to Traktor halftiming D&B tracks because "Automatic" BPM range chose the lower octave. `collection.nml` gitignored.

## Next

- **Strides workout loop builder** — for Saturday strides sessions: pick 4–6 strong drum-loop phrases from different tracks, loop each for a fixed duration (e.g. 20–30s on / 60s off × 6–10 reps), with silence/low-volume recovery gaps between reps. Candidates for phrase selection: reuse `traktor_beatgrid_ms` for bar-aligned cuts (4/8/16 bars at known BPM). Distinct from the continuous ramp-mix pipeline — more like an interval-timer with musical content. First step: pick a small pool of candidate tracks with clear drum-led intros and prototype a single-phrase loop before generalising.

## Open

- **Extension smart-fit** — the greedy picker sometimes lands well short of target (e.g. 28m → 33.7m) because any second pick would overshoot the upper tolerance. A two-pass or subset-sum picker that swaps a long pick for two shorter ones when the gap allows would hit target more reliably.
- **Consolidate off-target extension buckets** — extensions can land in 34/35/37/38min folders even when the intent was 36, because the slug derives from the actual total. Options: widen bucket rounding (e.g. nearest 5min), or expose a CLI flag to force the target bucket regardless of actual duration.
- **Lookup coverage** — residual ~41% of the DnB slice under 10min still lacks a trusted BPM source. Prioritise manual tap for frequently-picked artists over adding more scrapers.
- **Subgenre/vibe tagging** — add subgenre metadata to `tracks` (liquid, jump-up, neuro, minimal, halftime, rollers, etc.) so selection can filter by mood. Spike first: dump existing `genre` ID3 frames across the library to see what's already there vs. what needs hand-labelling. Then schema (likely a join table for many-to-many), backfill, and a `--subgenre` / `--exclude-subgenre` flag on the generator.
- **`/run-playlist` skill** — schedule-aware wrapper: reads today's row from the active training-plan file, parses distance → derives duration from easy pace, optionally asks subgenre preference, runs the generator. Path to the plan file is private — load from `.env` (e.g. `TRAINING_PLAN_GLOB=...this-week.md`) so the skill stays committable to the public repo. Confirm `.env` is gitignored before wiring.
- **Ratings + key-aware selection** — enrich `music.db` with iTunes/Apple Music ratings and favourites (track-level *and* artist-level — an artist favourite biases all their tracks up and pulls unrated tracks by that artist into the eligible pool), plus musical key (mixed-in-key / essentia / librosa). Use ratings to weight selection toward favourites and de-prioritise low-rated tracks; use key for harmonic adjacency across crossfades (Camelot wheel). Open question: rubberband preserves pitch so keys should survive retempo — verify before relying on it. Note: `traktor_key` is now populated for 1917 tracks (0–23 Open Key encoding) — translate to Camelot before wiring into selection.

## Later

- **Filler-track exclusion script** — `scripts/exclude_short_fillers.py` sets `run_exclude=1, run_exclude_reason='title_filler'` where `LOWER(title)` matches `%intro%`/`%outro%`/`%interlude%` AND `duration_s < 180`. Mirrors `analyse_beat_clarity.py` auto-exclusion. Current scan: 30 matches; duration threshold avoids false positives like LTJ Bukem — Cosmic Interlude (447s full-length).
- **Grid-aligned crossfades** — replace fixed-time crossfades in `generate_run_playlists.py` with phrase-aligned (bar or 16-bar) crossfades using `traktor_beatgrid_ms` + `traktor_bpm` as the anchor. Bigger audible quality win than upgrading the stretch algorithm. Tracks without Traktor data fall back to current behaviour.
- **Ableton/DawDreamer render path** — optional alternative to the ffmpeg+rubberband renderer. DawDreamer gives programmatic warping with no DAW dependency; generating an `.als` gives an inspectable intermediate artefact. Defer until/unless current renderer is a bottleneck.
- **M4A BPM tag write-back** — Traktor writes `TBPM` to mp3 but not `tmpo` to m4a (only `initialkey` lands). If downstream tools ever need file-level BPM on m4a, write it via mutagen from `traktor_bpm`. Not urgent — our pipeline reads `music.db`, not files.
